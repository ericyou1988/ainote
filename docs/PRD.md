# AInote 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
**AInote** — 融合 AI 提示词管理与英语学习的个人笔记应用。

### 1.2 产品定位
一款轻量级个人笔记工具，帮助用户随时记录 AI 提示词和英语句子，并通过 AI 自动拆解分析，把笔记变成学习材料。

### 1.3 目标用户
- **当前阶段**：单个用户（产品经理，AI 提示词和英语学习爱好者）
- **后续阶段**：开放注册，面向 AI 从业者和英语学习者

### 1.4 核心痛点
| 痛点 | 现状 | AInote 的解法 |
|------|------|-------------|
| 看到好的提示词想记录但缺乏结构化管理 | 散落各处，难以查找和复用 | 统一录入 + 标签分类 + 搜索 |
| 英文提示词看不懂、学不透 | 只能查单词，无法系统性理解 | AI 逐词翻译 + 语法树 + 纠错 |
| 提示词效果不好，不知道哪里需要优化 | 凭感觉修改 | AI 结构化拆解（角色/任务/输出格式）+ 改进建议 |
| 学完想进一步追问，但缺乏上下文 | 每次重新描述问题 | 笔记绑定独立对话线程，持久化可跨天追问 |

### 1.5 成功指标
- 用户能 10 秒内完成一条笔记录入
- AI 分析结果能帮助用户理解一条英文提示词的核心含义
- 用户愿意反复回到已保存的笔记进行追问学习

## 3. 功能需求

### 3.1 功能总览

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 笔记管理 | 录入、编辑、查看、删除、搜索、筛选、排序 | P0 |
| AI 分析 | 提示词结构化拆解、英文逐词翻译+语法树+纠错 | P0 |
| AI 对话 | 笔记绑定的独立对话线程 | P0 |
| AI 服务商 | 多服务商配置、切换、测试连接 | P0 |
| 标签系统 | 中文/英文 + 提示词/非提示词，多标签支持 | P0 |
| 语音朗读 | 英文内容语音播报 | P1 |
| 语音输入 | 语音转文字录入笔记 | P2（预留） |
| 笔记导出 | 导出为 Markdown/文本文件 | P1 |

### 3.2 首页/笔记列表

#### 3.2.1 页面描述
笔记的入口页面，以列表或卡片形式展示所有已保存的笔记。

#### 3.2.2 功能清单
| 功能 | 描述 | 交互细节 |
|------|------|---------|
| 笔记列表 | 展示笔记标题、标签药丸、更新时间、笔记状态 | 卡片或列表布局，按时间倒序。每条笔记左上角显示状态标签（未分析/已分析/有追问） |
| 新建笔记 | 点击右上角"+"按钮，跳转编辑器 | 空页面，标题+内容区 |
| 搜索 | 顶部搜索框，支持标题和内容关键词搜索 | 实时搜索，输入即搜 |
| 筛选 | 搜索框右侧筛选按钮，弹出标签筛选器 | 可选中文/英文/提示词标签，支持多选 |
| 排序 | 按时间/按标签排序 | 默认按更新时间倒序 |
| 下拉加载 | 长列表下拉加载更多笔记 | 首次加载 20 条，下拉追加 |
| 标签药丸 | 每条笔记显示关联的标签 | 圆角药丸形状，不同标签不同底色 |
| 笔记状态 | 列表中标注每条笔记的学习进度 | 未分析（灰色点）/ 已分析（蓝色点）/ 有追问（橙色点），方便快速定位需要继续学习的笔记 |
| AI 设置入口 | 页面底部有"设置"按钮 | 点击进入 AI 服务商管理页 |

#### 3.2.3 边界情况
- 无笔记状态：显示引导页，提示用户创建第一条笔记
- 搜索无结果：显示"未找到匹配的笔记"提示

### 3.3 笔记编辑器

#### 3.3.1 页面描述
全屏 Markdown 编辑器，用于创建和编辑笔记。

#### 3.3.2 功能清单
| 功能 | 描述 | 交互细节 |
|------|------|---------|
| 标题区 | 笔记标题输入，可展开/收起 | 默认展开，大号字体 |
| 标签栏 | 标题下方，支持添加/删除标签 | 点击"+"添加标签，选择预设（中文/英文/提示词）；点击"x"删除 |
| 工具栏 | 编辑区上方 | 加粗、斜体、标题、列表、链接、代码块 |
| 编辑区 | Markdown 内容输入 | 支持粘贴、打字录入 |
| AI 分析按钮 | 编辑区底部，醒目 | 点击后调用 AI 分析，显示加载动画和进度 |
| 保存 | 自动保存 + 手动保存 | 编辑过程中自动保存草稿，工具栏有保存按钮 |
| 导出 | 将笔记导出为本地文件 | 支持导出为 .md 和 .txt 格式，下载至用户本地设备 |
| 删除 | 删除笔记 | 弹出确认弹窗，防止误删 |

#### 3.3.3 边界情况
- 语言标签自动判断：系统根据笔记内容中中文/英文字符的比重自动打上语言标签。中文字符占比 > 50% 打"中文"标签，英文字符占比 > 50% 打"英文"标签。用户可手动修改
- 中英文混合且比重接近：同时打上中文和英文两个标签

#### 3.3.4 交互流程
```
用户点击新建 → 进入空编辑器 → 输入标题 → 输入内容 → 选择标签 → 自动保存 → 
用户可选择：
  - 点击"AI 分析" → 进入分析面板
  - 点击返回 → 回到首页
```

### 3.4 AI 分析面板

#### 3.4.1 页面描述
点击"AI 分析"后，展示 AI 对笔记内容的结构化拆解结果。

#### 3.4.2 AI 分析能力定义

AInote 内置两个 AI 专家角色，系统根据笔记内容类型自动选择或组合调用。

**专家 A：提示词工程师**
- 职责：结构化拆解提示词，评估提示词质量
- 分析维度：
  - 角色设定：识别提示词中 AI 角色定义是否清晰
  - 任务描述：识别核心任务目标是否明确
  - 输出格式：识别输出格式要求是否具体可执行
- 优化能力：给出具体的改进建议，并输出优化后的完整提示词版本
- 适用场景：笔记内容包含明确的指令/任务描述/输出要求

**专家 B：技术翻译专家**
- 职责：中英文双向翻译，技术术语精准处理
- 分析维度：
  - 逐词翻译表：每词的中英文对照、词性标注、语境说明
  - 语法树：句子结构分析（主句、从句、修饰关系）
  - 纠错建议：语法/拼写/表达地道性纠正
  - 技术术语：识别编程、AI、云计算等领域术语，确保翻译准确性
  - 改进版本：输出更地道/更精确的英文表达
- 适用场景：笔记内容为英文句子/段落，或中英混合内容

**自动内容识别规则：**
- 纯英文内容 → 调用技术翻译专家
- 纯中文内容且包含指令性语句 → 调用提示词工程师
- 中英混合 → 同时调用两位专家，翻译专家处理英文部分，提示词工程师处理中文指令部分

#### 3.4.3 功能清单
| 功能 | 描述 | 交互细节 |
|------|------|---------|
| 加载态 | 点击分析后显示进度 | 加载动画 + "AI 正在分析中..."提示 |
| 提示词分析 | 结构化拆解提示词 | 分三块卡片：角色设定 / 任务描述 / 输出格式 |
| 英文翻译分析 | 英语学习维度的拆解 | 逐词翻译表 / 语法树图 / 纠错建议 / 改进版本对比 |
| 折叠/展开 | 每块分析结果可折叠 | 默认展开，点击标题折叠 |
| 语音朗读 | 英文内容语音播报 | 分析面板右上角小喇叭图标，点击朗读英文内容 |
| 继续追问入口 | 面板底部按钮 | 点击跳转至 AI 对话面板 |

#### 3.4.3 边界情况
- **API Key 余额不足**：服务商返回余额不足错误时，显示"API 额度已用完，请检查服务商配置或更换服务商"，提供跳转到设置页的快捷链接
- **网络超时**：请求超过 30 秒未响应，显示"请求超时，请检查网络连接后重试"，支持点击"重新分析"
- **服务商不可用（5xx 错误）**：显示"服务商暂时不可用，请稍后重试"，支持切换其他已配置的服务商
- **API Key 无效（401/403）**：显示"API Key 无效或权限不足，请前往设置页检查"，提供跳转链接
- **内容过短**：笔记内容少于 10 个字符时，提示"内容太短，AI 无法有效分析"，不发起请求
- **非提示词、非英文内容**：AI 判断内容类型后选择对应分析模式；如果两者都不是，给出通用分析摘要

#### 3.4.4 提示词分析输出示例
```
角色设定：
  你是一位资深的数据科学家，专注于自然语言处理领域...

任务描述：
  请分析以下文本的情感倾向，并输出详细的评估报告...

输出格式：
  要求以 JSON 格式输出，包含以下字段：sentiment、confidence、key_phrases...

优化建议：
  - 建议增加具体的输入示例
  - 输出格式描述不够精确，建议明确字段类型
  - 可以增加约束条件防止 AI 跑题

改进版本：
  [展示优化后的完整提示词]
```

#### 3.4.5 英文分析输出示例
```
逐词翻译表：
  | Word | 翻译 | 词性 | 说明 |
  |------|------|------|------|
  | Act | 行动 | 动词 | 表示采取行动 |
  | as if | 仿佛 | 副词短语 | 引导虚拟语气 |
  | ... | ... | ... | ... |

语法树：
  主句 + 虚拟语气从句，整体为祈使句结构...

纠错建议：
  - "Act as if you are" → 建议使用 "Act as though you were"（虚拟语气更准确）

改进版本：
  [展示更地道/更精确的英文表达]
```

### 3.5 AI 对话面板

#### 3.5.1 页面描述
与笔记绑定的独立对话线程，用于针对分析结果进行追问。

#### 3.5.2 功能清单
| 功能 | 描述 | 交互细节 |
|------|------|---------|
| 对话历史 | 展示与当前笔记相关的所有对话消息 | AI 分析结果作为首条消息，用户回复和 AI 回复交替展示 |
| 发送消息 | 底部输入框 + 发送按钮 | 支持回车发送，支持多行文本 |
| 对话持久化 | 对话记录跨天保留 | 关闭页面后重新打开，对话仍在 |
| 语音输入入口 | 预留位置 | 灰色占位图标，显示"即将上线" |

#### 3.5.3 交互流程
```
用户在分析面板点击"继续追问" → 跳转对话面板（AI 分析结果已作为首条消息） → 
用户在底部输入问题 → 发送 → AI 回复 → 可继续追问
用户返回首页 → 稍后再次点击该笔记 → 进入对话面板 → 之前的对话仍在
```

### 3.6 AI 服务商设置

#### 3.6.1 页面描述
管理多个 AI 服务商配置，支持自由切换。

#### 3.6.2 功能清单
| 功能 | 描述 | 交互细节 |
|------|------|---------|
| 服务商列表 | 展示已添加的所有服务商 | 每条显示名称、API Key（脱敏显示如 sk-***abc）、接口地址、模型名称 |
| 添加服务商 | 添加新的 AI 服务商 | 表单：名称（必填）、API Key（必填）、接口地址（必填）、模型名称（必填） |
| 编辑服务商 | 修改已有配置 | 进入编辑模式，修改后保存 |
| 删除服务商 | 删除已有配置 | 弹出确认弹窗 |
| 启用/禁用 | 切换服务商状态 | 开关控件，禁用后该服务商不再可选 |
| 切换当前服务商 | 标记当前使用的服务商 | 当前激活的服务商用醒目标识（高亮背景或徽章） |
| 测试连接 | 验证配置是否生效 | 点击后向服务商发送一个测试请求，返回成功/失败提示 |

#### 3.6.3 边界情况
- API Key 无效（401/403）：测试连接失败时，提示"API Key 无效或权限不足，请检查后重试"
- 网络超时：请求超过 15 秒未响应，提示"连接超时，请检查网络或接口地址是否正确"
- 服务商不可用（5xx）：提示"服务商暂时不可用，请稍后重试"
- 无可用服务商：至少保留一个已启用的服务商，不允许全部禁用；当只剩一个服务商时，禁用开关置灰不可操作

## 4. 非功能需求

### 4.1 性能要求
- 首页加载时间 < 2 秒
- AI 分析响应时间取决于外部 AI 服务商（通常 3-10 秒），加载态需有进度反馈
- 笔记编辑自动保存，编辑过程中不丢内容

### 4.2 兼容性
- 浏览器：Chrome、Safari、Edge 最新版
- 移动端：iOS Safari、Android Chrome
- 桌面端：主流现代浏览器

### 4.3 数据安全
- AI 服务商的 API Key 在服务端加密存储
- 笔记数据不共享、不外传，仅用于 AI 分析请求
- 用户隐私：当前单用户，后续多用户时数据隔离

### 4.4 部署环境
- 部署在阿里云服务器
- 推荐配置：2C2G，Ubuntu 系统
- 使用 Nginx 做反向代理，HTTPS 访问

## 5. 设计规范

### 5.1 设计主题

### 5.2 色彩系统
| 用途 | 色值 | 说明 |
|------|------|------|
| 背景 | #FCF8FF | 暖白底色，减少视觉疲劳 |

### 5.5 间距
